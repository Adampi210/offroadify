# Product brief

## Product

**Offroadify**

## Initial user story

> As a user, I can upload a picture of a car and receive a realistic off-road-modified version that I can download.

## MVP flow

```text
Select image → Preview → Submit → Wait → View result → Download
```

## What the MVP validates

- Whether an image-editing API preserves the identity of the original car
- Whether one fixed prompt produces useful off-road edits
- Whether the upload/generation/download flow is viable
- Which failure modes should guide later research

## Not in the MVP

- Fine-grained modification controls
- Neutral backgrounds or masks
- Multiple views or 3D
- Custom model or local inference
- Accounts or saved projects

## Later directions

- Multiple modification presets
- Individual tire/suspension/rack/bumper/light controls
- Color changes
- Neutral backgrounds
- Mask-based editing
- Multi-view generation
- 3D generation and downloadable assets
- Custom-trained model
- End-user-device inference
